# 🌆 全天主报告 + 日概览生成 Prompt

## 任务

基于 inputs({DATE} 的 raw 采集),生成全天主报告**并同时输出当日 5-7 句概览**(prepend 到 summaries.md)。

## 数据范围

输入文件:
{INPUTS_LIST}

**报告范围**:{DATE} 当天 raw 中**全部 block**(00:00 - 23:59)。下面所有"今天""全天""今日"均指 {DATE} 当日内容,不要用你训练知识里的"完整 24h"先验。

## 定位

**这是用户真正用来定选题的主产出**。早间和午间是"不错过",晚间才是"决定做什么"。

需要严谨、完整、给出可执行建议。

---

## 两层维度

- **Layer 1 信息层(Q1-Q15)**:窗口内有什么有意思的信息(客观)
- **Layer 2 创作建议层(Q16-Q17)**:基于窗口内数据能写什么(主观 + 可执行)

---

## Layer 1 — 信息层(15 题)

> 下面所有"今天""当前"均指窗口内。

### 🌡️ Pulse 话题脉搏

**Q1. 窗口内最高声量事件**
窗口内跨多源同时出现的事件是什么?单点爆发不算,真正"出圈"的才算。

**Q2. KOL List 的辐射中心**
窗口内被 List 里最多人引用/转发的**单条推文**是什么?这往往是讨论原点。

**Q3. 产品 / 模型的深度发布**
窗口内有哪个 release 引发了技术性讨论(不是"好厉害"式转发)?评论区在争什么?

### ⚔️ Conflict 观点战场

**Q4. A vs B 的清晰对撞**
窗口内两方在同一问题上的公开分歧?各方核心论据?

**Q5. 反共识观点信号**
窗口内哪位 KOL 发了明确的反主流观点?论据站得住吗?

**Q6. 翻车 / 被打脸事件**
窗口内有谁被公开纠错 / 群嘲 / 道歉?

### 📖 Story 案例故事

**Q7. "某人用 AI 做了 X" 的故事**
窗口内最有故事相的个人案例?要有细节 + 反转 + 数字。

**Q8. 带真实数据的公开分享**
窗口内有谁公开了具体数字 —— 营收、用户数、token 消耗、转化率、A/B 结果?

**Q9. AI 辅助 / 取代工作的具体案例**
不是笼统"AI 会取代 X",要具体到"某工种 + 具体流程 + 具体结果"。

### 🛠️ Craft 实用干货

**Q10. 被多人验证的 prompt / 工作流**
窗口内有没有一个技巧被 2+ 个 KOL 独立验证有效?(教程稿素材)

**Q11. 适合"实测对比"的新品**
窗口内新出的模型/工具里,哪个做"我用它 vs 我用 X"横评会有流量?

**Q12. 可整理成清单的小技巧集**
窗口内零散但值得攒起来的小 tip?(攒到周末出清单稿)

### 🏢 Enterprise 企业落地信号

**Q-E. 解决企业实际问题的产品 / prompt / skill / 思想**
窗口内有没有哪个内容是在解决**具体企业场景**的问题?
(营销获客 / 降本提效 / 决策辅助 / 研发加速 / 客户服务)

这是用户独特的差异化定位(AI 博主 + 商业咨询),单独列出。

### 🔍 Edge 信号

**Q13. 出乎意料的冷 / 热反差**
窗口内什么东西"应该火却没火"或"不该火却火了"?反差本身就是钩子。

**Q14. 中英文温差**
英文圈热议但中文圈还没跟上的话题?(48h 抢首发窗口)

**Q15. 早期微弱信号**
窗口内讨论度不高但"气味对了"的论文 / 项目 / 观点?

## Layer 2 — 创作建议层(2 题)

### 🎯 Q16. 基于窗口数据的选题推荐(重点)

基于窗口内全部信号,结合用户的 3 种文章类型,推荐 1-3 个最优选题。

用户的 3 种文章类型:
1. **碎碎念**(猫笔刀式):日常盘点 + 大白话 + 微观点
2. **热点专题攻略**:切入点是"落地结合实际场景"
3. **AI 深度思考**:AI × 现实世界结合

对每个推荐选题给出:

```
### 选题 1: {一句话核心}
- **文章类型**: {碎碎念 / 攻略 / 深度}
- **推荐理由**: {为什么现在写这个,不是过几天}
- **差异化切入点**: {避开大路货,你的独特角度}
- **标题候选**:
  1. {标题 A}
  2. {标题 B}
  3. {标题 C}
- **核心钩子**: {一句话抓人点}
- **推荐开篇**: {2 句开头备选}
- **核心素材**: {来自窗口内哪 2-3 条推文 + 链接}
```

至少推 1 个,最多 3 个。不够就不硬凑。

### ⚠️ Q17. 窗口期预警

窗口内有没有"48 小时不写就过气"的选题?
(热度刚起 + 深度稿还没出现 + 有先发优势)

如有,独立列出,标红。

## 严格要求

## 双产出写入方式

evening 是全天总结,需要**同时输出两件事**(一次 LLM 调用,两次 Bash pipe):

### 1️⃣ 详细 wiki(走 wiki-write)

生成完整 markdown(上面"输出格式"那一份)后,Bash heredoc pipe:

```bash
{WIKI_WRITE_CMD} <<'PERCH_EOF'
(把上面"输出格式"里生成的完整 markdown 原样放这里,**不要**再包 `## section: {SECTION_NAME}` 外层标题,脚本会自动加)
PERCH_EOF
```

最终文件落在 `{WIKI_PATH}`(当日所有 section 共用一份)。

### 2️⃣ 5-7 句日概览(走 summary-write)

紧接着,提炼一条 5-7 句的日概览(用作 `{SUMMARIES_PATH}` 的 `## {DATE}` 索引条目):

```bash
{SUMMARY_WRITE_CMD} <<'PERCH_EOF'
(5-7 句概览正文,**不含** `## {DATE}` 标题,脚本自动加。覆盖:2-3 个最主要事件 / KOL 态度分布 / 关键数据 / 中英温差 / 异常信号。紧凑、信息密度高、不堆形容词,提到具体人/项目带 @handle。)
PERCH_EOF
```

最终条目 prepend 到 `{SUMMARIES_PATH}` 顶部(同日重跑替换前一次条目,幂等)。

**两步必须都跑**。少一步 = 当日 summary 缺失。**不要**用 Write 工具直接写 wiki / summaries,会破坏文件结构。
