# 🌅 早间简报生成 Prompt

## 任务

基于 `{RAW_PATH}`({DATE} 的 raw 采集),生成 **{SLOT}** 时段简报。

## 时间窗口(唯一真相)

- **类型**: {WINDOW_TYPE}
- **起点**: {WINDOW_START_LABEL}
- **终点**: {WINDOW_END_LABEL}

**严格按这个窗口过滤 raw**(block 标题行的 `MM-DD HH:MM` 字符串比较)。下面所有问题里出现"窗口内""本时段"等词,都指这个窗口 —— 不要用你记忆中的"早间=12h/15h/overnight"等先验。

**slot 定位**:morning 在 slots 里是首个 slot。配置为 `since_prev` 时运行时会 fallback 到 `today`(归属日 00:00 起);若你想覆盖"昨晚海外 overnight",在 `slots[0].start_hour` 之前跑 `report now` — 会 wrap 回昨日 evening,那份报告覆盖昨日完整 24h。本报告的时间语义永远以上面的 `{WINDOW_*}` 为准,不要假设窗口长度。

## 聚焦题目(早间版 7 题)

早间只关注跟得上节奏需要的 7 题,不做全量分析。所有"窗口内"都指上面定义的窗口。

### Q1. 最高声量事件

窗口内跨 3+ 数据源同时出现的事件是什么?
(数据源 = 多位 KOL 独立转发/评论、原推高 RT 数、被引用讨论)

输出:事件一句话 + 信号强度标注 + 核心 3 方观点 + Takeaway

### Q2. KOL List 的辐射中心

窗口内被 List 里最多人引用/转发的**单条推文**是什么?
这往往是当前讨论的"原点",值得顺着扒完整语境。

输出:原推内容 + 辐射范围(被谁转/评) + 为什么重要

### Q3. 重要产品 / 模型发布

窗口内有哪个产品或模型的 release 引发了技术性讨论(非"好厉害"式转发)?

输出:发布内容 + 评论区的专业争论点 + Takeaway(是否值得做技术解读)

### Q6. 翻车 / 被打脸事件

窗口内有谁被公开纠错 / 群嘲 / 道歉?
如无,此题可跳过。

### Q14. 中英文温差(海外视角)

窗口内英文圈热议但**中文圈还没跟上**的话题?
这是中文博主 48 小时抢首发的窗口。

输出:话题描述 + 英文声量 + 建议中文化切入角度 + 预估窗口剩余时间

### Q15. 早期微弱信号

窗口内讨论度还不高但"气味对了"的论文/项目/观点?
(AK 推荐的新 arxiv、小众但高质量的技术文、冷门但可能爆的想法)

输出:项目/观点 + 为什么有潜力 + Takeaway(是否值得做前瞻稿)

### Q17. 窗口期预警

综合以上,今天有没有"48 小时不写就过气"的选题?

## 写作风格

- **紧凑**:每题 3-6 行,不啰嗦
- **数据优先**:有具体数字(RT/评论/star)就标出来
- **带链接**:推文、项目必带 source URL,便于你点开看原文
- **不重复**:如果 Q1 和 Q3 聊的是同一件事,后者指向前者即可

## 判定规则

- **Signal 强度**:🟢🟢🟢 = 跨 3+ 源独立验证 / 🟢🟢 = 2 源 / 🟢 = 1 源但有独立判断价值 / ⚪ = 数据不足
- **Takeaway 不能水**:写不出具体建议的题,老实写"无特殊行动建议"
- **窗口外的推文不要分析**,即使它看起来很相关 — 违反就是不遵守 prompt 的时间窗口
