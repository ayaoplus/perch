# 🌅 早间简报生成 Prompt

## 任务

基于 `{RAW_PATH}`（{DATE} 的当日 raw），生成当日早间简报。

## 时间窗口

- **类型**: {WINDOW_TYPE}
- **起点**: {WINDOW_START_LABEL}
- **终点**: {WINDOW_END_LABEL}

按 raw block 标题行的 `MM-DD HH:MM` 时间戳过滤,只分析落在此窗口的推文。

**morning 是首个 slot** — `since_prev` 会按 DESIGN §3.1 简化 fallback 到 `today`(归属日 00:00 起),**不跨昨日 raw**。如果当前窗口覆盖不到你想要的"海外 overnight",两种可选:
1. 在凌晨 05:00 前跑 `node scripts/report.mjs now --topic ai-radar` — 会 wrap 回昨日 evening,那份报告以"归属日完整 24h"视角覆盖昨晚 + 昨日全天
2. 如果 raw 文件时段确实不足,按实际可用数据生成,在报告里标注"数据窗口不完整"

## 聚焦题目（早间版 7 题）

早间只关注跟得上节奏需要的 7 题，不做全量分析。

### Q1. 海外 overnight 最高声量事件

过去 12-15h，跨 3+ 数据源同时出现的事件是什么？
（数据源 = 多位 KOL 独立转发/评论、原推高 RT 数、被引用讨论）

输出：事件一句话 + 信号强度标注 + 核心 3 方观点 + Takeaway

### Q2. KOL List 的辐射中心

过去 12-15h 被你 List 里最多人引用/转发的**单条推文**是什么？
这往往是今晚的"讨论原点"，值得顺着扒完整语境。

输出：原推内容 + 辐射范围（被谁转/评） + 为什么重要

### Q3. 重要产品 / 模型发布

过去 12-15h 有哪个产品或模型的 release 引发了技术性讨论（非"好厉害"式转发）？

输出：发布内容 + 评论区的专业争论点 + Takeaway（是否值得做技术解读）

### Q6. 翻车 / 被打脸事件

过去 12-15h 有谁被公开纠错 / 群嘲 / 道歉？
如无，此题可跳过。

### Q14. 中英文温差（海外视角）

过去 12-15h 英文圈热议但**中文圈今早还没跟上**的话题？
这是中文博主 48 小时抢首发的窗口。

输出：话题描述 + 英文声量 + 建议中文化切入角度 + 预估窗口剩余时间

### Q15. 早期微弱信号

过去 12-15h 讨论度还不高但"气味对了"的论文/项目/观点？
（AK 推荐的新 arxiv、小众但高质量的技术文、冷门但可能爆的想法）

输出：项目/观点 + 为什么有潜力 + Takeaway（是否值得做前瞻稿）

### Q17. 窗口期预警

综合以上，今天有没有"48 小时不写就过气"的选题？

## 写作风格

- **紧凑**：每题 3-6 行，不啰嗦
- **数据优先**：有具体数字（RT/评论/star）就标出来
- **带链接**：推文、项目必带 source URL，便于你点开看原文
- **不重复**：如果 Q1 和 Q3 聊的是同一件事，后者指向前者即可

## 判定规则

- **Signal 强度**：🟢🟢🟢 = 跨 3+ 源独立验证 / 🟢🟢 = 2 源 / 🟢 = 1 源但有独立判断价值 / ⚪ = 数据不足
- **Takeaway 不能水**：写不出具体建议的题，老实写"无特殊行动建议"
